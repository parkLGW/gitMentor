# GitMentor 测试指南

## 快速开始

### 方式1: 本地开发模式

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 方式2: 加载构建版本

```bash
# 构建项目
npm run build

# 生成的文件在 dist/ 目录中
```

## 在Chrome中加载扩展

### 步骤1：打开Chrome扩展管理页面
1. 在Chrome地址栏输入：`chrome://extensions/`
2. 或通过菜单：☰ > 更多工具 > 扩展程序

### 步骤2：启用开发者模式
- 点击右上角的 "开发者模式" 开关

### 步骤3：加载未打包的扩展
- 点击 "加载未打包的扩展"
- 选择项目的 `dist/` 文件夹
- 完成！

## 功能测试清单

### 📋 概览标签 (Overview Tab)

**测试项目**：推荐使用这些著名项目进行测试

| 项目 | 链接 | 预期难度 |
|------|------|--------|
| React | https://github.com/facebook/react | 高级 |
| Vue | https://github.com/vuejs/core | 中级 |
| Express | https://github.com/expressjs/express | 初级 |
| Next.js | https://github.com/vercel/next.js | 中级 |

**测试步骤**：
1. 在Chrome中打开任意GitHub项目页面
2. 点击右上角的GitMentor图标
3. 查看概览卡片内容

**验证点**：
- [ ] 显示项目名称和所有者
- [ ] 获取项目描述
- [ ] 显示Star、Fork、Issues数量
- [ ] 显示最后更新时间
- [ ] 显示学习难度等级（初级/中级/高级）
- [ ] 显示项目活跃状态
- [ ] 识别已存档项目（如果存在）
- [ ] 列出解决的问题
- [ ] 列出适用场景
- [ ] 支持大数字缩写（如1.5k）

### 🚀 快速上手标签 (Quick Start Tab)

**测试步骤**：
1. 在概览标签验证后，点击"快速上手"标签
2. 查看前置知识、安装步骤、示例代码和常见问题

**验证点**：
- [ ] 显示前置知识列表
- [ ] 列出安装步骤（代码块）
- [ ] 显示第一个示例代码
- [ ] 显示可展开的常见问题
- [ ] 点击问题可以展开/折叠解决方案
- [ ] 代码块使用暗色主题展示

### 🗺️ 源码地图标签 (Source Map Tab)

**测试步骤**：
1. 点击"源码地图"标签
2. 查看项目架构、关键文件、学习路径等内容

**验证点**：
- [ ] 显示项目架构概述
- [ ] 列出关键文件和优先级
- [ ] 文件优先级使用颜色编码（红/黄/蓝）
- [ ] 显示推荐阅读顺序
- [ ] 学习阶段可展开/折叠
- [ ] 列出关键概念
- [ ] 显示有用的学习提示

### 🌍 多语言支持

**测试步骤**：
1. 在插件右上角找到语言切换按钮
2. 切换为"中文"和"英文"

**验证点**：
- [ ] UI元素正确切换为对应语言
- [ ] 所有标签页内容使用正确语言
- [ ] 语言设置持久化（刷新后保持）
- [ ] 中文项目自动检测为中文
- [ ] 英文项目自动检测为英文

### ⚡ 性能测试

**测试步骤**：
1. 打开大型项目（如React、Django）
2. 观察加载时间和内存使用

**验证点**：
- [ ] 插件启动 < 1秒
- [ ] 数据加载 < 3秒
- [ ] API调用有缓存（重复访问快速）
- [ ] 没有明显的内存泄漏

### 🐛 错误处理

**测试场景**：

1. **非GitHub页面**
   - 测试步骤：打开Google、GitHub非项目页面
   - 预期：显示"请在GitHub项目页打开此插件"

2. **API限制**
   - 测试步骤：快速打开多个项目标签（模拟API限制）
   - 预期：显示友好的错误消息，可重试

3. **网络错误**
   - 测试步骤：断网后打开插件
   - 预期：显示错误消息，如果有缓存则显示缓存数据

## 手动测试用例

### 用例1: 新用户评估项目

```
场景：开发者想快速了解是否应该学习一个新项目
步骤：
1. 在GitHub搜索找到目标项目
2. 打开项目页面
3. 点击GitMentor插件
4. 在概览标签中快速扫一眼（应该在30秒内）
5. 查看"解决的问题"和"适用场景"
6. 根据学习难度决定是否继续

预期时间：5-10分钟内完成决策
```

### 用例2: 新手快速上手

```
场景：初学者想快速安装和运行项目
步骤：
1. 打开项目的GitMentor
2. 点击"快速上手"标签
3. 按照前置知识检查清单
4. 复制安装步骤到终端
5. 运行示例代码
6. 查看常见问题解决方案

预期效果：30分钟内成功运行项目
```

### 用例3: 源码学习规划

```
场景：开发者想深入学习项目源码
步骤：
1. 打开"源码地图"标签
2. 理解项目架构
3. 按照优先级查看关键文件
4. 按照推荐顺序规划学习路径
5. 理解关键概念

预期效果：有清晰的学习计划和文件阅读顺序
```

## 已知限制

### MVP版本限制

1. **无实时项目结构分析**
   - 当前使用模板化的源码地图
   - 后续版本可添加真实的GitHub API项目树分析

2. **README解析简单**
   - 使用基础的正则表达式匹配
   - 某些复杂的README格式可能无法正确解析
   - 后续版本可集成NLP或LLM进行智能解析

3. **无高级代码分析**
   - 当前版本未实现深度代码解析（用户触发式）
   - 计划在V1.1中添加

### 浏览器兼容性

- ✅ Chrome 100+
- ⚠️ Edge (基于Chromium，应该支持)
- ❌ Firefox (需要不同的适配)
- ❌ Safari (需要不同的适配)
